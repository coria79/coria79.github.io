Recipe to Connect GitHub with SSH from Linux

Introduction:

As a developer, it will be necessary to get used to the Linux environment. One of the required tasks will be to connect to a remote repository such as GitHub. To connect securely, one of the available methods is by using an SSH key.

Objective:

To show the necessary steps to successfully connect to a remote repository through an SSH key to authenticate the user.

Ingredients:

- Ubuntu Linux
- Terminal
- Browser
- Active GitHub account

Cooking time:

- 10 to 15 minutes.

Instructions:

- Open the terminal.
- Open the browser.
- In the browser, sign in to your GitHub account.
- In the terminal, run the following commands.

Update the operating system.
sudo apt update && sudo apt upgrade -y

Check if Git is installed.
git --version

If it is not installed, run the following command to install it.
sudo apt install git

Verify that you are in the user’s home directory.
pwd

Enter the hidden .ssh folder.
cd .ssh

Generate the SSH key.
ssh-keygen

It will ask you to enter the file name.
Since you are in the same folder as the SSH keys, there is no need to type the full path.
id_ed25519_test
Press Enter

It will ask for a passphrase to increase security.
In this case, it will not be necessary — just press Enter.
It will ask to confirm the passphrase again — just press Enter.

The result of the SSH key creation will be displayed.
Your identification has been saved in /home/your_user/.ssh/id_ed25519_test
Your public key has been saved in /home/your_user/.ssh/id_ed25519_test.pub

Verify that the files are in the folder:
id_ed25519_test ← private key (do not share)
id_ed25519_test.pub ← public key (copied to GitHub)

Display the content of the public key.
cat id_ed25519_test.pub

Copy the entire line displayed and paste it into your GitHub account.
In the browser, go to your GitHub account.
Click your avatar → “Settings” → “SSH and GPG keys”.
Click New SSH key.
Title: Linux Key
Key: paste the copied content — only the public key.
Click Add SSH key.


In the terminal, execute the following command:
ssh -T git@github.com

When asked whether to trust the host, type "yes" and press Enter.
If everything went well, you will see the following:
Hi <your_user>! You've successfully authenticated

Choose a working folder, for example:
cd Documents
Clone the repository with:
git clone git@github.com:<YOUR_USER>/<YOUR_REPO>.git

Final comments:

The private key must remain only on the local machine and must not be shared with anyone. Generating the SSH key allows you to connect securely with the repository.
