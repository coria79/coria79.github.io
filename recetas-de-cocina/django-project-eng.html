For Programmers

Recipe for creating a project in Django.

Ingredients:

- Web browser.
- Terminal.

Cooking Time:

- 5 to 10 minutes.

Prerequisites:

- Completed the recipe: "Django Framework on Ubuntu from Windows with virtual machines."

Objective:

The use of the Linux terminal is an almost essential tool for developers. Using the terminal will be a daily routine. Therefore, it's important to develop the habit from the beginning. A good practice would be to type the commands directly rather than copy-pasting. This will greatly help in incorporating the use of the terminal and entering commands.

Instructions:

- Open the terminal. Access the terminal by pressing "Ctrl + Alt + T" or searching for "Terminal" in the applications menu.
- Know the current folder using the "pwd" command. It will show something like: /home/user, where "user" is the username configured during installation. In this case, we are in the user's home directory.
- Use the "ls" command to see the contents of the folder. In this case, you will see the contents of the "user" folder, something like: Desktop Documents Downloads Music Pictures Public snap Templates Videos.
- Create the main project folder within the "Documents" folder.
- Use the command "cd Documents" to enter this folder.
- Create the main folder that will contain different Djando projects using the command "mkdir projects_django".
- Enter the newly created folder using "cd projects_django".
- Use the already installed "virtualenv" virtual environment. Create the virtual environment with the command "virtualenv -p python3 env-my_project".
- Activate the virtual environment with the command "source env-my_project/bin/activate".
- You can confirm the activation of the virtual environment because the prompt will show: (env-my_project).
- Install Django with the command "pip install django".
- Create the project using the command "django-admin startproject my_project".
- Use the "ls" command to show the folders inside "projects_django"; you should see the newly created project folder "my_proyect".
- The output of the "ls" command should be: manage.py my_project.
- Enter this folder with the command "cd my_project".
- Create the app with the command "python manage.py startapp my_app".
- Note that the app's name must be different from the project's name; otherwise, an error will occur.
- Move up one folder level with the command "cd ..".
- Execute the project with the command "python manage.py runserver".
- Open the browser, enter the URL: http://localhost:8000.

The recipe for the basic installation of the Django Framework has been completed.

Final Comments:

The use of the terminal can be done with the Linux terminal or with the terminal built into the IDE. In this case, as it is the first project, the Linux terminal was chosen. However, the majority of usage might be within the IDE, depending on each developer's comfort.

The virtual environment was configured in a folder outside the Django project but with a name similar to the project. This is done to make it more intuitive to recognize the virtual environment and its corresponding project. Different projects may have different virtual environments, for example, a virtual environment working with an older version of Python. This separation of folders is done intuitively.

Opening the browser and going to: localhost:8000 is a way to verify that the Django installation is correct. You will see a generic Django welcome page.
