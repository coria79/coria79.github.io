Recipe for creating a Django project.

Objective:

Perform the necessary tasks to create a project in Django and finalize settings in the development environment.

It is divided into two sections. By completing all the sections, you will have what is needed to start development.

Recipe to start a Django project:


Ingredients:

- Web browser.
- Terminal.

Cooking Time:

- 10 to 15 minutes.

Prerequisites:

- Completed the recipe: "Django Framework on Ubuntu from Windows with virtual machines."

Objective:

The use of the Linux terminal is an almost essential tool for developers. Getting used to using the terminal will be a daily occurrence. Therefore, it is important to develop the habit from the beginning. A good practice would be to type the command directly rather than copy-pasting. This will help a lot in incorporating the use of the terminal and entering commands.

Instructions:

- Open the terminal. Access the terminal by pressing "Ctrl + Alt + T" or searching for "Terminal" in the application menu.
- Know the current folder using the "pwd" command.
- It will show something similar to: /home/user, where "user" is the username configured during installation. In this case, we are in the home of the user.
- Use the "ls" command to see the contents of the folder. In this case, you will see the contents of the "user" folder.
- It will show something similar to: Desktop Documents Downloads Music Pictures Public snap Templates Videos
- Create the main project folder inside the "Documents" folder.
- Use the "cd Documents" command to enter this folder.
- Create the main folder that will contain the different Django projects using the "mkdir projects_django" command.

Activate Virtual Environment:

- Enter the newly created folder using "cd projects_django".
- Use the virtual environment already installed "virtualenv".
- Use the "virtualenv -p python3 env-my_project" command to create the virtual environment.
- Use the "source env-my_project/bin/activate" command to activate the virtual environment.
- You can confirm the activation of the virtual environment because the prompt will show: (env-my_project)

Install Django:

- Use the "pip install django" command to install Django.

Create Project:

- Use the "django-admin startproject my_project" command to create the project.
- Use the "ls" command to show the folders inside "projects_django". You should see the newly created project folder "my_proyect".
- The output of the "ls" command should be: manage.py my_project
- Use the "cd my_project" command to enter this folder.

Create Application:

- Use the "python manage.py startapp my_app" command to create the app.
- One detail to keep in mind is that the app's name must be different from the project's name. If you try to create it with the same name, you will get an error.

Run Server:

- Use the "cd .." command to go up one level of the folder.
- Use the "python manage.py runserver" command to run the project.
- Open the browser, enter the URL: http://localhost:8000 or its alternative http://127.0.0.1:8000
- CONTROL + C (to end the server execution)

The recipe for the basic installation of the Django Framework has been completed.

Final Comments:

The use of the terminal can be done with the Linux terminal. But it can also be done with the terminal that comes built into the IDE itself. In this case, since it is the first project, we chose to use the Linux terminal. But most likely, the majority of usage will be within the IDE, although it depends on each developer's comfort level.

The virtual environment was configured in a folder outside the Django project but with a name similar to the project. This is done to make it a bit more intuitive to recognize the virtual environment and the corresponding project. Since there may be different projects with different virtual environments, for example, a virtual environment that works with an earlier version of Python. This intuitively separates the folders.

Opening the browser and entering: localhost:8000 is the way to verify that the Django installation is correct. You will see a generic welcome page from Django.

Regarding the use of English. Since it is the language on which all programming languages are based and on which all documentation is done, this site will follow the rule that everything will be in English except what needs to be shown to the user, which will be in Spanish (or another language). That's why you see that even the folder names will be in English.

Recipe for finalizing development environment settings:


Open Project Folder With PyCharm:

- Start PyCharm.
- The welcome window appears (Open).
- Open the window to browse folders, select the project folder (OK).
- It will ask us if we trust the contents of the folder, check.
"Trust projects in ~/Documents/django_projects/my_project (Trust Project).
- The project's folder tree will open.
- In the "Tool Windows Bar," click on the "Terminal" icon (at the bottom of the bar). This bar is located just to the left of the browser panel (where the project's folders and files are visible).
- The prompt will be visible, for example:
user@user-virtual-machine:~/Documents/projects_django/my_project$

Open Project Folder With Visual Studio Code:

- Launch Visual Studio Code.
- The welcome window will appear.
- For the first time, it will prompt you to choose the "theme"; select "Dark Modern".
- Go to the File menu -> Open Folder...
- A window will open for navigating folders; choose the project folder, for example:
Documents/projects_django/my_project (Open)
- It will ask if you trust the content of the folder; check:
"Trust the authors of all files in the parent folder 'my_project' (Yes, I trust the authors)

Activate the virtual environment:

- Enter the following command to go up one level in the folders: cd ..
- Use the "ls" command to validate that you are in the indicated folder; you should see the env-my_project folder.
- source env-my_project/bin/activate
- (env-my_project)

Perform Migrations and Migrate:

- Enter the following command to enter the project folder: cd my_project
- Use the "ls" command to validate that you are in the indicated folder; you should see the manage.py file.
- Enter the following command to perform migrations: python manage.py makemigrations
- The default output would be: No changes detected.
- Enter the following command to migrate: python manage.py migrate
- The console will show the operations to be performed. Operations to perform: Apply all migrations: admin, auth, contenttypes, sessions
- Pending migrations that are by default when a Django project starts will be performed.
- Optional: In case there are no migrations to perform, the console will show the message: No migrations to apply.

Select the interpreter from PyCharm:

- Main menu "File" -> "Settings..."
- The Settings window will open.
- On the left side, different options will be displayed.
- Click on "Project: my_project" (here you will see the name of the project folder you created).
- In the main part of the window, click on "Python Interpreter."
- In "Python interpreter," the Python version will be displayed, and then the interpreter's path will be displayed. It needs to be changed.
- Click on "Add Interpreter."
- Click on "Add Local Interpreter..."
- In "Environment," check "Existing."
- Click on "..."
- Navigate through the folders until you are in:
/home/user/Documents/projects/env-my_project/bin
- In this case, being in the "bin" folder, select "python3.10 (OK).
- In "Interpreter," the complete path of the interpreter that was just configured will be displayed (OK).
- In "Python interpreter," you will see the interpreter with the folder we created when creating the virtual environment.

Add Database Connector:

SQLite is the default database used by Django. The following steps are to develop with this database engine.
- Without closing the settings window.
- You can see the "+" and "-" buttons.
- Select "+"
- In the search box, start entering the text "sqlalchemy-pysqlite3-binary."
- Select it from the list.
- Click "Install Package."
- A small notice will appear at the bottom of that same window: "Package 'sqlalchemy-pysqlite3-binary' installed successfully."
- Close the window.
- The previous window with the newly installed package can be viewed (OK).

DBeaver:

SQLite is the default database used by Django. The following steps are to connect to this database engine.
- Start the dbeaver-ce application.
- It may show "Tip Of The Day" (Close).
- Click on the "New Database Connection" icon.
- The assistant opens, select "SQLite" (Next).
- Check "Host."
- (Open...) To indicate the path where the sqlite3 file is located, in this case:
user/documents/django_projects/my_project
- Select the file. The file's name is: db.sqlite3 (Select).
- (Test Connection...) To validate that DBeaver can successfully connect to the database.
- Optional: It may be necessary to download drivers (Download).
- It will show that the connection result was successful (OK).
- Finish the database connection wizard (Finish).
- The DBeaver window should remain open; it can be minimized.
