Receta para crear proyecto en Django.

Objetivo:

Realizar las tareas necesarias para crear un proyecto en Django y las configuraciones finales en el entorno de desarrollo.

Se encuentra dividido en dos secciones. Al completar todas las secciones se podrá contar con lo necesario para iniciar el desarrollo.

Receta para iniciar proyecto Django:


Ingredientes:

- Navegador web.
- Terminal.

Tiempo De Cocción:

- 10 a 15 minutos.

Requisitos Previos:

- Haber completado la receta: "Framework Django en Ubuntu desde Windows con máquinas virtuales."

Objetivo:

El uso de la terminal de Linux es una herramienta casi imprescindible como desarrollador. Será cotidiano el uso de la terminal. Por eso es importante ir haciéndose el hábito desde el principio. Una buena práctica sería tomarse el trabajo de tipear directamente el comando antes que hacer copiar-pegar. Eso ayudará mucho a incorporar el uso de la terminal y de ingresar comandos.

Instrucciones:

- Abre la terminal. Acceder a la terminal presionando "Ctrl + Alt + T" o buscando "Terminal" en el menú de aplicaciones.
- Conocer la carpeta en la que te encuentras actualmente por medio del comando "pwd".
- Se mostrará algo similar a: /home/user, donde "user" es el nombre de usuario configurado al realizar la instalación. En este caso estamos en el home de user.
- Por medio del comando "ls" se pude conocer el contenido de la carpeta. En este caso se verá el contenido de la carpeta "user".
- Se mostrará algo similar a: Desktop Documents Downloads Music Pictures Public snap Templates Videos
- Se desea crear la carpeta principal del proyecto dentro de la carpeta "Documents".
- Por medio del comando "cd Documents" se ingresa a esta carpeta.
- Crear la carpeta principal que contendrá los diferentes proyectos de Djando por medio del comando "mkdir projects_django".

Activar Ambiente Virtual:

- Ingresar a la carpeta que se acaba de crear por medio de "cd projects_django".
- Se utilizará el entorno virtual que ya ha sido instalado "virtualenv".
- Por medio del comando "virtualenv -p python3 env-my_project" se creará el entorno virtual.
- Por medio del comando "source env-my_project/bin/activate" se activará el entorno virtual.
- Se puede confirmar la activación del entorno virtual porque en el prompt se puede ver: (env-my_project)

Instalar Django:

- Por medio del comando "pip install django" se instala Django.

Crear Proyecto:

- Por medio del comando "django-admin startproject my_project" se crea el proyecto.
- Por medio del comando "ls" mostrar las carpetas que se encuentran dentro de "proyectos_django" debería verse la carpeta del proyecto recién creado "my_proyect".
- La salida del comando "ls" debería ser: manage.py my_project
- Por medio del comando "cd my_project" ingresar en esta carpeta.

Crear Aplicación:

- Por medio del comando "python manage.py startapp my_app" se crea la app.
- Un detalle a tener en cuenta es que el nombre de la app debe ser diferente al nombre del proyecto. Si se intenta crear con el mismo nombre se obtendrá un error.

Ejecutar Servidor:

- Por medio del comando "cd .." se sube un nivel de carpeta.
- Por medio del comando "python manage.py runserver" se ejecuta el proyecto.
- Iniciar el navegador, colocar la url: http://localhost:8000 o su alternativa http://127.0.0.1:8000
- CONTROL + C (finaliza la ejecución del servidor)

Se ha completado la receta para la instalación básica del Framework Django.

Comentarios finales:

El uso de la terminal puede realizarse con la terminal de Linux. Pero también puede realizarse con la terminal que trae incorporado el mismo IDE. En este caso al ser el primer proyecto se optó por usar la terminal de Linux. Pero muy posiblemente el mayor uso sea dentro del IDE, aunque depende de la comodidad de cada desarrollador.

Se configuró el entorno virtual en una carpeta por fuera del proyecto Django pero con un nombre similar al proyecto. Esto es así para que sea un poco más intuitivo reconocer el entorno virtual y el proyecto que le corresponde. Ya que es posible que hayan diferentes proyectos con diferentes entornos virtuales, por ejemplo un entorno virtual que trabaje con una versión anterior de Python. De esta forma se separan de forma intuitiva las carpetas.

Iniciar el navegador e ingresar a: localhost:8000 es la forma de comprobar que la instalación de Django es correcta. Se verá una página de bienvenida genérica de Django.

Respecto del uso del inglés. Al ser el idioma sobre el que se basan para la creación de todos los lenguajes de programación y sobre el que se realizan todas las documentaciones. En este sitio se mantendrá la regla que será todo en inglés excepto lo que deba mostrarse al usuario que deba ser en español (u otro idioma). Por eso se ve que hasta los nombres de las carpetas serán en inglés.

Receta para realizar los ajustes finales del entorno de desarrollo:


Ingredientes:

- PyCharm / Visual Studio Code.
- DBeaver.

Tiempo De Cocción:

- 10 a 15 minutos.

Objetivo:

Realizar configuraciones necesarias sobre el IDE y sobre el gestor de base de datos para dejar listo el entorno de desarrollo.

Abrir Carpeta De Proyecto Con Pycharm:

- Iniciar PyCharm.
- Se muestra ventana de bienvenida (Open).
- Se abre ventana para navegar las carpetas, seleccionar la carpeta del proyecto (OK).
- Nos consultará si confiamos en el contenido de la carpeta, tildar.
"Trust projects in ~/Documents/django_projects/my_project (Trust Project).
- Se abrirá el árbol de carpetas del proyecto.
- En la "Tool Windows Bar" hacer click sobre el ícono de "Terminal" (hacia la parte inferior de la barra). Esta barra se encuentra justo del lado izquierdo del panel de navegador (donde se ven las carpetas y los archivos del proyecto).
- Se verá el prompt, por ejemplo:
user@user-virtual-machine:~/Documents/projects_django/my_project$

Abrir Carpeta De Proyecto Con Visual Studio Code:

- Iniciar visual studio code.
- Se mostraá ventana de bienvenida.
- Al ser la primera vez solicitará elegir el "theme", elegir "Dark Modern".
- Menú File -> Open Folder...
- Se abre ventana para navegar las carpetas, seleccionar la carpeta del proyecto la ruta por ejemplo sería:
Documents/projects_django/my_project (Open)
- Nos consultará si confiamos en el contenido de la carpeta, tildar:
"Trust the authors of all files in the parent folder 'my_project' (Yes, I trust the authors)

Activar en entorno virtual:

- Ingresar el siguiente comando para subir un nivel en las carpetas: cd ..
- Por medio del comando ls se valida que se encuentra en la carpeta indica, se debe ver la carpeta env-my_project
- source env-my_project/bin/activate
- (env-my_project)

Realizar las migraciones y migrate:

- Ingresar el siguiente comando para ingresar en la carpeta del proyecto: cd my_project
- Por medio del comando ls para validar que se encuentra en la carpeta indicada, es necesario ver el archivo manage.py
- Ingresar el siguiente comando para realizar la migraciones: python manage.py makemigrations
- La salida por defecto sería: No changes detected.
- Ingresar el siguiente comando para realizar el migrate: python manage.py migrate
- Se mostrará por consola las operaciones a realizar. Operations to perform: Apply all migrations: admin, auth, contenttypes, sessions
- Se realizarán las migraciones pendientes que son por defecto apenas se inicia un proyecto en Django.
- Opcional: En el caso que no haya alguna para realizar se verá por consola el mensaje: No migrations to apply.

Seleccionar el intérprete desde Pycharm:

- Menú principal "File"-> "Settings..."
- Se abrirála venta a de Settings.
- Del lado izquierdo se mostrarán diferentes opciones.
- Hacer click en "Project: my_project" (aquí se verá el nombre de la carpeta del proyecto que hemos creado).
- En la porción principal de la ventana hacer click en "Python Interpreter".
- Se mostrará en "Python interpreter" la versión de python y luego a continuación se muestra la ruta donde se encuentra ese intérprete. Es necesario cambiarlo.
- Hacer click en "Add Interpreter".
- Hacer click en "Add Local Interpreter..."
- En "Enviroment" tildar "Existing".
- Hacer click en "..."
- Navegar las carpetas hasta estar en:
/home/user/Documents/projects/env-my_project/bin
- Y este caso estando en la carpeta "bin" seleccionar "python3.10 (OK).
- En "Interpreter" se mostrará la ruta completa del intérprete que se acaba de configurar (OK).
- En "Python interpreter" se verá el intérprete con la carpeta que nosotros hemos creado al crear en entorno virtual.

Agregar conector de la base de datos:

SQlite es la base de datos por defecto que utiliza Django. Los siguientes pasos son para poder desarrollar con este motor de base de datos.
- Sin haber cerrado la ventana de setting.
- Se puede ver botón de "+" y "-".
- Seleccionar "+"
- En el recuadro de búsqueda comenzar a ingresar el texto "sqlalchemy-pysqlite3-binary".
- Seleccionarlo del listado.
- Botón "Install Package".
- Se mostrará un pequeño aviso en la parte inferior de esa misma ventana: "Package 'sqlalchemy-pysqlite3-binary'installeded successfully".
- Cerrar ventana.
- Se podrá visualizar la ventana anterior con el paquete recién instalado (OK).

Seleccionar el intérprete desde Visual Studio Code:
(Próximamente)

DBeaver:

SQlite es la base de datos por defecto que utiliza Django. Los siguientes pasos son para poder conectarse con este motor de base de datos.
- Iniciar la aplicación dbeaver-ce.
- Posiblemente muestre "Tip Of The Day" (Close).
- Hacer click sobre el ícono "New Database Connection".
- Se abre asistente, seleccionar "SQLite" (Next).
- Dejar tildado "Host".
- (Open...) Para indicar la ruta en la que se encuentra el archivo sqlite3, en este caso es:
user/documents/django_projects/my_project
- Seleccionar el archivo. El nombre del archiv es: db.sqlite3 (Select).
- (Test Connection...) Para validar que DBeaver se puede conectar de forma exitosa a la base de datos.
- Opcional: Es posible que sea necesario descargar controladores (Download).
- Se mostrará que el resultado de la conexión ha sido exitosa (OK).
- Finalizar el asistente de conexión a la base de datos (Finish).
- La ventana de DBeaver debe permanecer abierta, se puede minimizar.

Se ha completado la receta para realizar los ajustes finales del entorno de desarrollo
