Recetas De Cocina
Para Programadores

Receta para crear proyecto en Django.

Ingredientes:

- Navegador web.
- Terminal.

Tiempo De Cocción:

- 5 a 10 minutos.

Requisitos Previos:

- Haber completado la receta: "Framework Django en Ubuntu desde Windows con máquinas virtuales."

Objetivo:

El uso de la terminal de Linux es una herramienta casi imprescindible como desarrollador. Será cotidiano el uso de la terminal. Por eso es importante ir haciéndose el hábito desde el principio. Una buena práctica sería tomarse el trabajo de tipear directamente el comando antes que hacer copiar-pegar. Eso ayudará mucho a incorporar el uso de la terminal y de ingresar comandos.

Instrucciones:

- Abre la terminal. Acceder a la terminal presionando "Ctrl + Alt + T" o buscando "Terminal" en el menú de aplicaciones.
- Conocer la carpeta en la que te encuentras actualmente por medio del comando "pwd".
- Se mostrará algo similar a: /home/user, donde "user" es el nombre de usuario configurado al realizar la instalación. En este caso estamos en el home de user.
- Por medio del comando "ls" se pude conocer el contenido de la carpeta. En este caso se verá el contenido de la carpeta "user".
- Se mostrará algo similar a: Desktop Documents Downloads Music Pictures Public snap Templates Videos
- Se desea crear la carpeta principal del proyecto dentro de la carpeta "Documents".
- Por medio del comando "cd Documents" se ingresa a esta carpeta.
- Crear la carpeta principal que contendrá los diferentes proyectos de Djando por medio del comando "mkdir projects_django".
- Ingresar a la carpeta que se acaba de crear por medio de "cd projects_django".
- Se utilizará el entorno virtual que ya ha sido instalado "virtualenv".
- Por medio del comando "virtualenv -p python3 env-my_project" se creará el entorno virtual.
- Por medio del comando "source env-my_project/bin/activate" se activará el entorno virtual.
- Se puede confirmar la activación del entorno virtual porque en el promtp se puede ver: (env-my_project)
- Por medio del comando "pip install django" se instala Django.
- Por medio del comando "django-admin startproject my_project" se crea el proyecto.
- Por medio del comando "ls" mostrar las carpetas que se encuentran dentro de "proyectos_django" debería verse la carpeta del proyecto recién creado "my_proyect".
- La salida del comando "ls" debería ser: manage.py my_project
- Por medio del comando "cd my_project" ingresar en esta carpeta.
- Por medio del comando "python manage.py startapp my_app" se crea la app.
- Un detalle a tener en cuenta es que el nombre de la app debe ser diferente al nombre del proyecto. Si se intenta crear con el mismo nombre se obtendrá un error.
- Por medio del comando "cd .." se sube un nivel de carpeta.
- Por medio del comando "python manage.py runserver" se ejecuta el proyecto.
- Iniciar el navegador, colocar la url: http://localhost:8000

Se ha completado la receta para la instalación básica del Framework Django.

Comentarios finales:

El uso de la terminal puede realizarse con la terminal de Linux. Pero también puede realizarse con la terminal que trae incorporado el mismo IDE. En este caso al ser el primer proyecto se optó por usar la terminal de Linux. Pero muy posiblemente el mayor uso sea dentro del IDE, aunque depende de la comodidad de cada desarrollador.

Se configuró el entorno virtual en una carpeta por fuera del proyecto Django pero con un nombre similar al proyecto. Esto es así para que sea un poco más intuitivo reconocer el entorno virtual y el proyecto que le corresponde. Ya que es posible que hayan diferentes proyectos con diferentes entornos virtuales, por ejemplo un entorno virtual que trabaje con una versión anterior de Python. De esta forma se separan de forma intuitiva las carpetas.

Iniciar el navegador e ingresar a: localhost:8000 es la forma de comprobar que la instalación de Django es correcta. Se verá una página de bienvenida genérica de Django.

Respecto del uso del inglés. Al ser el idioma sobre el que se basan para la creación de todos los lenguajes de programación y sobre el que se realizan todas las documentaciones. En este sitio se mantendrá la regla que será todo en inglés excepto lo que deba mostrarse al usuario que deba ser en español (u otro idioma). Por eso se ve que hasta los nombres de las carpetas serán en inglés.
